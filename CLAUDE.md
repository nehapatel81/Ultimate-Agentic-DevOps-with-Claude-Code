# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a **static portfolio website** for the DevOps Micro Internship (DMI) program. It demonstrates how to deploy a static website to AWS using modern infrastructure as code (Terraform) and CI/CD automation (GitHub Actions).

**Stack:**
- Frontend: HTML5 + CSS3 (no build step)
- Infrastructure: AWS S3 + CloudFront
- IaC: Terraform
- CI/CD: GitHub Actions (auto-deploy on push to main)
- Auth: OIDC role assumption (no stored AWS credentials)

## Architecture

```
Local Development (Edit HTML/CSS)
        ↓
Push to main branch
        ↓
GitHub Actions Workflow
        ├─→ Checkout code
        ├─→ Assume AWS role via OIDC
        ├─→ Sync files to S3 (pravinmishradmi-site-production)
        └─→ Invalidate CloudFront cache
        ↓
Site live at CloudFront URL
```

**Infrastructure Components (deployed via Terraform):**
- **S3 Bucket:** Stores static HTML/CSS/image files
- **CloudFront Distribution:** CDN serving the site globally, configured for index.html as root object
- **IAM Role:** `github-actions-deploy` (OIDC-enabled for GitHub Actions)

## Deployment Workflow

### First-Time Setup

1. **Scaffold Terraform infrastructure:**
   ```
   /scaffold-terraform eu-north-1 pravinmishradmi
   ```
   This generates S3, CloudFront, and IAM configurations.

2. **Review the plan:**
   ```
   /tf-plan
   ```

3. **Apply infrastructure:**
   ```
   /tf-apply
   ```

4. **Deploy site files:**
   ```
   /deploy
   ```

### Ongoing Deployments

- **Automatic:** Any push to `main` branch triggers GitHub Actions (see `.github/workflows/deploy.yml`)
- **Manual:** Use `/deploy` skill to sync and invalidate cache

### Manual AWS CLI Deployment (if needed)

```bash
# Get infrastructure outputs
cd terraform && terraform output -json

# Sync files (replace bucket-name and dist-id with terraform outputs)
aws s3 sync . s3://bucket-name \
  --delete \
  --exclude "terraform/*" \
  --exclude ".git/*" \
  --exclude ".github/*" \
  --exclude "*.md" \
  --exclude ".claude/*"

# Invalidate CloudFront
aws cloudfront create-invalidation --distribution-id dist-id --paths "/*"
```

## Development Notes

### No Build Step Required
This is a static site—just edit HTML/CSS directly. No bundler, minifier, or transpiler needed.

### AWS Configuration
- **Region:** eu-north-1 (EU North)
- **S3 Bucket:** `pravinmishradmi-site-production` (or your custom name)
- **CloudFront Distribution ID:** Retrieved from Terraform outputs
- **IAM Role ARN:** `arn:aws:iam::533267262133:role/github-actions-deploy` (OIDC-enabled)

### Testing Changes Locally
1. Open `index.html` directly in a browser (file:// protocol) to preview
2. Check `style.css` for responsive design
3. Push to main when satisfied—GitHub Actions will deploy

### Common Edits

**Edit portfolio content:**
- Modify HTML directly in `index.html`
- Update images in `images/` directory
- Customize styling in `style.css`

**Change deployment target:**
- Re-run `/scaffold-terraform` with different region/project name
- Update AWS credentials or IAM role in `.github/workflows/deploy.yml` if needed

## Troubleshooting

**CloudFront returning old content?**
- The deploy skill automatically invalidates cache (`/*`)
- CloudFront invalidation takes ~1 minute to propagate

**Files not syncing to S3?**
- Check S3 bucket name in Terraform outputs: `terraform output -json | grep bucket`
- Verify AWS credentials and IAM permissions
- Ensure `.github/workflows/deploy.yml` has correct role ARN

**Terraform plan shows unexpected changes?**
- Run `/tf-plan` to review what will change before applying
- If resources look wrong, destroy and regenerate: `terraform destroy` then `/scaffold-terraform`
